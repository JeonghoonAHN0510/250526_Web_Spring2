[ ORM ] Object Relational Mapping(객체 관계 매핑)
- 객체지향 프로그래밍에서 사용되는 객체와 DB(관계형 데이터베이스) 매핑(연결) 기술
- 편의성 제공
- 사용처 : MyBatis, JPA 등등

[ Hibernate ]
- Java에서 ORM을 구현한 라이브러리
- SQL 없이 순수 Java만으로 데이터베이스를 처리할 수 있다.
- 기능
    1. CRUD
    2. 타입변환
    3. 검색, 정렬, 페이징처리 등의 부가기능

[ JPA ] Java Persistence Api
- Java에서 ORM을 사용할 때 사용되는 표준 API
- 자바에서 다양한 라이브러리 제공(+ Hibernate)
- 설치 : gradle에 implementation 'org.springframework.boot:spring-boot-starter-d ata-jpa' 추가
- 기능 : 영속된 관계에서 자바 객체가 변화되면, 데이터베이스도 변경된다.

[ Entity ]
- 데이터베이스 테이블과 매핑된 클래스
- Entity클래스 = DB테이블, Entity객체 = DB레코드
- 클래스 : 객체의 설계도
    class Member {
        int id;
        String name;
    }
- 객체 : 클래스의 인스턴스
    new Member(1, "유재석");
- 사용법
    1. 선언 : 해당 클래스 위에 @Entity 주입한다.
    -> Entity 1개당 @Id 1개는 필수(primary key)
    2. Entity 매핑 설정 : application.properties
    -> Entity 생성 관리 : spring.jpa.hibernate.ddl-auto
        1) create : Entity 테이블을 삭제 후 생성
        2) update : Entity 테이블이 존재하면 수정, 존재하지 않으면 생성
        3) none(기본값) : 기능 X
    -> JPA가 처리한 SQL을 콘솔에 표시
        1) spring.jpa.show-sql=true : 표시 활성화
        2) spring.jpa.properties.hibernate.format_sql=true : 줄바꿈 처리
        3) logging.level.org.hibernate.sql=DEBUG
           logging.level.org.hibernate.orm.jdbc.bind=TRACE : SQL 매개변수를 표시

[ JPA Repository ]
- 기본 CRUD 작업과 동적쿼리를 제공하는 인터페이스
- 사용법
    1. Repository 인터페이스 생성
    2. extends JpaRepository<T, ID>
        T : 조작할 Entity 클래스명
        ID : 조작할 Entity의 PK의 자료형, 기본자료형은 불가능
    3. 인터페이스 위에 @Repository를 주입한다.

[ JPA 기본함수 ]
1. .save(저장할 Entity) : 저장할 Entity의 PK가 존재하지 않으면 INSERT, 존재하면 UPDATE
2. .findAll() : 모든 Entity를 조회
3. .findById() : 특정 Entity를 조회 -> Optional로 반환
4. .deleteById() : 특정 Entity를 삭제
- 수정함수는 별도로 존재하지 않는다. -> 영속성의 특징 때문에
-> save, findAll, findById 등등 반환된 Entity가 영속된 상태이다.
-> @Transactional을 갖는 클래스 또는 메소드 내의 .setter 함수를 이용하여 수정처리
-> 관례적으로 Entity는 MVC 패턴 내에서는 Service에서만 사용된다.

[ Optional ]
- NULL과 관련된 메소드 기능을 제공하는 클래스
1. .isPresent() : null이면 false, 아니면 true 반환
2. .get() : 해당 객체 반환
3. .orElse(null일때 값) :
4. .orElseThrow(예외객체) : 예외객체를 던짐
